## 技术亮点

### 1. 并发控制 🔒
使用 Redis 分布式锁处理座位并发预约，防止超卖：
```typescript
async decrementAvailableSeats(routeId: number, seats: number): Promise<boolean> {
  const lockKey = `route:lock:${routeId}`;
  const acquired = await this.redisService.acquireLock(lockKey, 5000);
  if (!acquired) {
    throw new HttpException('系统繁忙', HttpStatus.TOO_MANY_REQUESTS);
  }
  // ... 扣减座位
  await this.redisService.releaseLock(lockKey);
}
```

### 2. 事务处理 💾
订单创建使用数据库事务保证原子性：
```typescript
async create(passengerId: number, dto: CreateOrderDto): Promise<Order> {
  const queryRunner = this.dataSource.createQueryRunner();
  await queryRunner.connect();
  await queryRunner.startTransaction();
  try {
    // 创建订单 + 扣减座位
    await queryRunner.commitTransaction();
  } catch (error) {
    await queryRunner.rollbackTransaction();
    throw error;
  }
}
```

## 已知问题和解决方案

### 问题 1: 小程序需要 HTTPS 域名 ⚠️
**影响**: 生产环境必须使用 HTTPS 域名

**解决方案**:
- 开发阶段：在微信开发者工具中勾选"不校验合法域名"
- 生产环境：配置 HTTPS 证书和域名

### 问题 2: OSS 存储未实现 ⚠️
**影响**: 在职证明图片无法上传

**临时方案**:
- 使用 Base64 编码临时存储
- 或手动配置本地文件上传

**最终方案**:
- 集成腾讯云 COS
- 实现图片上传和水印处理

### 问题 3: 缺少单元测试 ⚠️
**影响**: 代码质量无法保证，重构风险高

**计划**:
- 优先为核心服务编写单元测试
- 订单流程编写集成测试
- 配置 CI 自动运行测试
