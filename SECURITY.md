# 安全政策

## 报告安全漏洞

我们非常重视安全问题。如果你发现了安全漏洞，请通过以下方式联系我们：

- 发送邮件至项目维护者
- 或者创建一个私密 Issue

**请勿公开披露安全漏洞**，以便我们有时间在影响用户之前修复问题。

## 安全建议

1. **不要在代码中硬编码密钥** - 使用环境变量或配置文件
2. **定期更新依赖** - 使用 `npm audit` 检查已知漏洞
3. **保护 JWT_SECRET** - 使用强随机字符串
4. **HTTPS** - 生产环境必须使用 HTTPS
5. **数据库访问控制** - 限制数据库访问权限
